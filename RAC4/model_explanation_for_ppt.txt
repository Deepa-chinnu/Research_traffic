SLIDE 1: Models Overview

Title: "Machine Learning Models Evaluated"

Content:

SLIDE 2: Linear Models
----------------------
Title: "Linear Models (Baseline Approaches)"

1. Linear Regression
   • Basic linear relationship: y = β₀ + β₁x₁ + ... + βₙxₙ
   • Fast training, highly interpretable
   • Limitation: Cannot capture nonlinear traffic patterns

2. Ridge Regression
   • L2 regularization: Penalizes large coefficients (λ∑βᵢ²)
   • Handles multicollinearity well
   • Fastest training (0.001s) - ideal baseline
   • Result: R² = 1.000, Utility = 0.999897

3. Lasso Regression
   • L1 regularization: Automatic feature selection (λ∑|βᵢ|)
   • Eliminates irrelevant features
   • Result: R² = 1.000, good for feature importance

4. ElasticNet
   • Combines L1 + L2 regularization
   • Balances feature selection and stability
   • Result: R² = 1.000, Utility = 0.999815

SLIDE 3: Nonlinear Models
-------------------------
Title: "Nonlinear Models"

1. Support Vector Regression (SVR)
   • RBF Kernel: Captures nonlinear relationships using radial basis functions
   • Linear Kernel: Faster but linear only
   • Memory efficient (uses support vectors)
   • Result: Good performance but requires tuning

2. K-Nearest Neighbors (KNN)
   • Instance-based: Predicts using k nearest neighbors (k=3, 5, 11)
   • No training phase (lazy learning)
   • Result: R² ≈ 0.99, fast training (0.002s)
   • Limitation: Slow prediction, sensitive to irrelevant features

3. Decision Tree
   • Tree-based: Creates binary splits based on feature values
   • Highly interpretable (can visualize tree structure)
   • Result: R² = 0.999990, zero sensitivity
   • Limitation: Prone to overfitting

SLIDE 4: Ensemble Methods
--------------------------
Title: "Ensemble Methods (Best Performers)"

1. Random Forest
   • Type: Ensemble of decision trees (bagging)
   • Architecture variants: 50, 100, 200 trees
   • Key features:
     - Averages predictions from multiple trees
     - Reduces overfitting through bagging
     - Handles missing values naturally
     - Provides feature importance
   • Results:
     - 50 trees: R² = 0.999994, Utility = 0.983 (BEST)
     - 100 trees: R² = 0.999995, Utility = 0.972
     - 200 trees: R² = 0.999995, Utility = 0.947
   • Why best: High accuracy + Fast training (0.24s) + Low sensitivity (ΔR² < 1e-6)

2. Extra Trees (Extremely Randomized Trees)
   • Similar to Random Forest but more randomization
   • Random splits instead of optimal splits
   • Result: R² = 0.999999, fastest ensemble training (0.087s)
   • Utility = 0.993891

3. Gradient Boosting
   • Sequential ensemble (boosting)
   • Builds trees sequentially, each correcting previous errors
   • Result: R² = 0.999917, Utility = 0.927
   • Limitation: Slower training (1.037s), more sensitive to tuning

SLIDE 5: Neural Networks

Title: "Neural Networks"

SLIDE 7: Why Random Forest Wins
--------------------------------
Title: "Random Forest: Optimal Choice for Traffic Prediction"

Advantages:
• Handles nonlinear traffic dynamics (congestion × capacity × incidents)
• Fast training (0.24s) and inference - suitable for real-time systems
• Low sensitivity to architectural changes (ΔR² < 1e-6) - robust to tuning
• Handles missing values, outliers, and mixed feature types naturally
• Strong Utility Score (0.983) - best composite metric
• Production-ready: Stable defaults, easy tuning, explainable feature importances
• Captures complex interactions that linear models cannot

Comparison:
• vs. Ridge: Random Forest captures nonlinear patterns (Ridge is linear only)
• vs. Gradient Boosting: Random Forest is faster and less sensitive
• vs. Extra Trees: Random Forest has better Utility Score
• vs. Decision Tree: Random Forest reduces overfitting through ensemble

SLIDE 8: Model Selection Criteria
----------------------------------
Title: "Evaluation Metrics & Model Selection"

1. Accuracy Metrics:
   • R² Score: Coefficient of determination (higher is better)
   • RMSE: Root Mean Squared Error (lower is better)
   • MAPE: Mean Absolute Percentage Error (lower is better)

2. Efficiency Metrics:
   • Training Time: Seconds to train model (lower is better)
   • Prediction Speed: Time for inference (critical for real-time)

3. Robustness Metrics:
   • Sensitivity: ΔR² when architecture changes (lower is better)
   • Indicates stability to parameter tuning

4. Composite Metric:
   • Utility Score = 0.7·R² + 0.2·(1 - TrainTime/max) + 0.1·(1 - Sensitivity)
   • Balances all three aspects: Accuracy (70%), Speed (20%), Robustness (10%)

Result: Random Forest achieves highest Utility Score (0.983) among all models.
